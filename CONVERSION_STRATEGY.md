## âš¡ Speed Optimization Strategy

### Critical Rendering Path
1. **Inline critical CSS** â€” Emergency bar, header, and hero styles should be inlined in `<head>` for production
2. **Font preconnect** â€” Google Fonts connected early via `<link rel="preconnect">`
3. **Hero image eager loading** â€” `loading="eager"` on hero image, `lazy` on everything else
4. **System font fallback** â€” `-apple-system, BlinkMacSystemFont` in font stack

### Image Optimization
1. **WebP format** â€” Convert all images to WebP with PNG fallback
2. **Responsive images** â€” Use `srcset` for mobile/tablet/desktop variants
3. **Compression** â€” Target 80% quality for hero (large), 70% for gallery (small)
4. **Dimensions declared** â€” `width` and `height` on all `<img>` tags prevent layout shift

### JavaScript Performance
1. **IntersectionObserver** â€” Used for animations instead of scroll listeners
2. **Passive scroll listeners** â€” `{ passive: true }` for header shadow
3. **requestAnimationFrame** â€” Counter animations use rAF for 60fps
4. **Deferred loading** â€” Script at end of `<body>`, no blocking

### WordPress-Specific Recommendations
1. **Caching plugin** â€” WP Super Cache or W3 Total Cache
2. **Image CDN** â€” ShortPixel or Imagify for automatic WebP + compression
3. **Minification** â€” Autoptimize for CSS/JS minification and concatenation
4. **Hosting** â€” Managed WordPress (WP Engine, Flywheel) for server-level caching
